Por dentro

A extensão é o servidor.

O portal não tem backend na nuvem. Uma extensão do VS Code sobe um servidor HTTP em 127.0.0.1, serve a interface React e faz proxy para o Copilot e para os MCPs — usando as credenciais que o VS Code já tem.

Os três pacotes do repositório

PacoteResponsabilidade
extension/Extensão VS Code: servidor HTTP/SSE, loop agêntico, ferramentas, MCPs e storage. É o único processo com acesso a disco e rede.
web/Interface React (Vite), servida pela própria extensão como assets estáticos. Estado de UI em Zustand + localStorage.
shared/Tipos TypeScript compartilhados — o contrato da API entre os dois lados.

O caminho de uma mensagem

  1. O navegador manda a mensagem via REST e abre um stream SSE para a resposta.
  2. A extensão monta o prompt: instruções do projeto/agente, skills ativas, arquivos fixados no contexto, bases de conhecimento e histórico (com resumo automático quando a conversa não cabe na janela do modelo).
  3. O loop agêntico chama o modelo via vscode.lm. Se a resposta pede ferramentas (modo Agent), a extensão executa cada tool call — arquivo, comando, busca, MCP — e devolve os resultados ao modelo, em rodadas, até a resposta final.
  4. Cada token gerado desce pelo SSE em tempo real. O stream é reconectável: se o navegador cair no meio, a resposta continua no servidor e a UI retoma de onde parou.
  5. Tudo é persistido em disco a cada passo — fechar o navegador não perde nada.

Modelo de segurança

  • Só localhost. O servidor escuta em 127.0.0.1 e exige um token gerado na instalação — outra máquina da rede não alcança nem autentica.
  • Sandbox de arquivos. Toda operação de arquivo resolve o caminho dentro da pasta de trabalho da conversa e rejeita escapes (.., caminhos absolutos, symlinks não autorizados). Pastas referenciadas são a exceção controlada: só os alvos que o usuário registrou.
  • Checkpoints. Antes de qualquer escrita/edição/remoção o portal tira um snapshot — reverter é um clique no card da ferramenta.
  • Segredos no SecretStorage. Tokens de MCP, cookies e client_secrets ficam no cofre do VS Code, cifrados em repouso — nunca em JSON no disco.
  • Sem telemetria própria. O portal não coleta nem envia dados de uso.

Onde ficam os dados

layout do disco
~/AIChatPortal/            # config global da máquina
  config.json              # porta, token, proxy/CA, projectsRoot
  runtime.json             # porta/pid enquanto o servidor está de pé

<dados do portal>/         # portal-data/ (ou ~/AIChatPortal)
  projects/<projeto>/      # arquivos do projeto + .aiportal/ (meta)
  sessions/                # conversas persistidas (JSON)
  workspaces/              # pastas das conversas avulsas
  skills/  knowledge/      # skills e bases de conhecimento
  bmad/                    # instalação global do BMAD Method
Consequência prática: backup e portabilidade são copiar pastas. Nenhum dado do seu trabalho vive fora do seu disco.
