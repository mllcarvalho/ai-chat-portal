Integrações

MCPs: as ferramentas do assistente.

MCP (Model Context Protocol) é o padrão aberto que conecta assistentes de IA a sistemas reais — consultar a AWS, abrir chamados, ler bases internas. No portal, os MCPs viram ferramentas que o assistente usa sozinho no modo Agent, com liga/desliga por servidor e por conversa.

De onde vêm os servidores

OrigemComo entra
mcp.json do VS Code Os servidores que você já configurou (incluindo os do projeto em .vscode/mcp.json) aparecem automaticamente, com liga/desliga pela UI.
Setup guiado do portal ConsumerLab e IUClick têm assistentes de instalação passo a passo dentro do portal (detalhes abaixo).
Proxy OAuth2 Gateways MCP corporativos (client_credentials): o portal obtém o token e conecta via Streamable HTTP. O client_secret fica no SecretStorage do VS Code, cifrado em repouso.

ConsumerLab — dados de consumo na AWS

O MCP do ConsumerLab conecta o assistente aos dados do laboratório na AWS. O setup guiado replica o fluxo manual inteiro, sem terminal:

  1. Pré-requisitos — confere git, Python e o gerenciador uv (o bootstrap da instalação já deixa tudo pronto).
  2. Repositório — clona o servidor MCP e instala as dependências (uv sync), passando pelo proxy corporativo.
  3. Login AWS SSO — você escolhe o portal SSO (Landing Zone ou CTPRO, cada um com a própria região), a conta e a role. Máquinas onde o navegador não abre têm fallback por código de dispositivo.
  4. Registro — o portal grava o profile AWS no ~/.aws/config e registra o servidor stdio. A conta conectada fica visível na tela do MCP.
Senha expirou ou trocou de conta? Clique na linha do ConsumerLab e rode o setup de novo — ele sobrescreve a configuração anterior.

IUClick — chamados do ServiceNow

O MCP do IUClick permite consultar e movimentar chamados sem sair da conversa. O setup guiado:

  1. Pré-requisitos — confere Node e npx.
  2. Registry — grava o registry privado (Artifactory) no ~/.npmrc e valida o pacote @ai-stack-fn7/mcp-servers.
  3. Credenciais — Cookie e X-UserToken são opcionais: quando informados, ficam no SecretStorage do VS Code (cifrados) e entram como variáveis na subida do servidor. Sem eles, a autenticação é feita pela tool login na própria sessão. O portal também consegue capturar os cookies do navegador automaticamente.
  4. Registro — registra o servidor stdio (npx … service-now --stdio).

Rede corporativa: o que o portal resolve por você

  • Proxy e CA herdados. Todo MCP stdio sobe com o proxy corporativo e a CA interna já injetados no ambiente — a mesma configuração da tela de login do portal.
  • Segredos fora do disco. Tokens e cookies vivem no SecretStorage do VS Code, nunca em arquivos de texto.
  • Timeout com progresso. Ferramentas MCP demoradas têm 5 minutos de timeout, com reset a cada notificação de progresso — consultas longas não caem no meio.
  • Diagnóstico primeiro. Se um MCP não conecta, a tela de Diagnóstico testa a conectividade e diz exatamente o que está faltando (proxy, CA, registry, uv…).
